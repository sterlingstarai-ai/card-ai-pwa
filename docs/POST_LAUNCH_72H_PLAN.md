# Post-Launch 72H Plan

Card AI v1.0.0 출시 후 72시간 운영 계획

---

## 타임라인 개요

| 시간 | 단계 | 주요 활동 |
|------|------|----------|
| 0-4H | 즉시 모니터링 | 크래시, API 오류 감시 |
| 4-24H | 안정화 | 초기 이슈 대응 |
| 24-48H | 피드백 수집 | 사용자 리뷰, 제보 분석 |
| 48-72H | 평가 | KPI 리뷰, 핫픽스 계획 |

---

## 1. 이슈 트리아지 규칙

### P0 - Critical (즉시 대응)
**정의:** 서비스 전체 불가 또는 데이터 손실

| 증상 | 예시 | 대응 시간 |
|------|------|----------|
| 앱 크래시 | 시작 시 즉시 종료 | 1시간 이내 |
| 데이터 손실 | 저장된 카드 사라짐 | 1시간 이내 |
| 보안 이슈 | 민감 정보 노출 | 즉시 |
| API 완전 장애 | 모든 요청 실패 | 1시간 이내 |

**대응 절차:**
1. 즉시 Vercel 롤백 (이전 안정 버전)
2. 원인 분석 (로그, Sentry)
3. 핫픽스 개발
4. 테스트 후 재배포
5. 사후 분석 문서 작성

### P1 - High (4시간 이내)
**정의:** 핵심 기능 장애, 일부 사용자 영향

| 증상 | 예시 | 대응 시간 |
|------|------|----------|
| 데모 모드 실패 | 데모 진입 불가 | 4시간 |
| 카드 추가 불가 | 저장 실패 | 4시간 |
| 제보 기능 장애 | Issue 미생성 | 4시간 |
| 특정 기기 크래시 | iOS 17에서만 발생 | 4시간 |

**대응 절차:**
1. 이슈 재현 확인
2. 임시 해결책 (킬스위치 등)
3. 근본 수정
4. 테스트 후 배포

### P2 - Medium (24시간 이내)
**정의:** 부가 기능 문제, UX 불편

| 증상 | 예시 | 대응 시간 |
|------|------|----------|
| UI 깨짐 | 특정 해상도에서 레이아웃 틀어짐 | 24시간 |
| 검색 품질 저하 | 검색 결과 부정확 | 24시간 |
| 데이터 오류 | 혜택 정보 틀림 | 24시간 |
| 성능 저하 | 로딩 느림 | 24시간 |

**대응 절차:**
1. 이슈 등록
2. 우선순위 태그
3. 다음 릴리즈에 포함

### P3 - Low (다음 릴리즈)
**정의:** 개선 사항, 사소한 버그

| 증상 | 예시 |
|------|------|
| 오타 | 텍스트 수정 |
| 개선 제안 | UX 개선 요청 |
| 스타일 | 색상, 폰트 조정 |

## 3. 핵심 KPI

### 안정성 지표 (목표)
| 지표 | 목표 | 알림 기준 |
|------|------|----------|
| 크래시율 | < 0.5% | > 2% |
| API 성공률 | > 99% | < 95% |
| 앱 시작 시간 | < 3초 | > 5초 |

### 사용성 지표 (참고)
| 지표 | 관찰 |
|------|------|
| 데모 완료율 | 온보딩 → 데모 완료 비율 |
| 카드 등록율 | 앱 설치 → 카드 1개 이상 등록 |
| 제보 수 | 일평균 제보 건수 |

## 6. 킬스위치 (긴급 비활성화)

특정 기능에 문제 발생 시 `src/constants/config.js`에서 비활성화:

```javascript
FEATURES: {
  MAP_ENABLED: false,      // 지도 기능 OFF
  OCR_ENABLED: false,      // OCR 기능 OFF
  LOCATION_ENABLED: false, // 위치 기능 OFF
}
```

변경 후 재배포:
```bash
git add src/constants/config.js
git commit -m "hotfix: disable [기능] feature"
git push && vercel --prod
```
